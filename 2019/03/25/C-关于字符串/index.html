Posted on 2019-03-25 | In 技术

字符串一直都是一个让我头疼的问题。因为许多算法题目，需要输入输出的地方多多少少都会涉及到字符串。

在这里，让我和字符串做一个了断吧。如果以后在涉及到字符串头疼的问题，可能就是 需要我自己做一个word吧。嘻嘻嘻

从哪里 讲起呢？
本想从输入-输出讲起，但cin,cout我也说不什么新花样。还是从字符串开始讲起吧~ 参照C++ Primer Plus。

C风格字符串

cat可以说是一个字符串，因为末尾有\0结尾。而dog 则谈不上字符串，只能称为 数组。
在cout打印的时候，cat到\0的时候就停止打印了；而dog则一直打印，把内存中后面的各个字节解释为字符打印出来，遇到\0 停止。

隐式的在末尾加了\0。这个叫做 字符串常量 或 字符串字面值 （使用””）(而字符常量使用’’)

第一个是 将 83 赋值给 shirt_size。
第二个 “S” 不是字符常量，表示 字符’S’和\0. 而且”S” 实际上是字符串所在的内存地址。

srelen() 与 sizeof()

这对难兄难弟本来没有多大关系，但是…放在一起 就让人 混乱。
strlen()只计算可见字符，不把空字符计算在内。 若字符串A 用数组存储，则数组的长度不能小于 strlen(A)+1 。
而 sizeof() 是这个变量大小，不在意里面有什么东西，是不是为空，就是占地大小。

一个常规操作

声明一个字符数组，可以直接用cin 进行赋值操作，前提是不能越界。

cin
